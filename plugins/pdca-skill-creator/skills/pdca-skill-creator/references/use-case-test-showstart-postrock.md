# ShowStart Post-Rock Plugin Use Case

## 用例目标

验证 `pdca-skill-creator` 生成的爬虫/分类/插件型 PDCA 技能，是否能从秀动列表页批量采集多条演出，覆盖安装、运行、字段提取、业务分类、网络诊断、dry-run 同步和交付清洁。

## 用户需求摘要

- 从秀动列表页 `https://www.showstart.com/` 抓取演出卡片，而不是只抓取一个固定详情页。
- 第一轮每次至少采集 `5` 条去重演出记录；`max_items` 必须可配置，推荐默认 `20`，不得把数量写死为一条。
- 列表采集先提取演出名、艺人/演出主体、详情页 URL、列表顺序和卡片证据；字段缺失必须逐条诊断，不得整批伪造成功。
- 先在多条结果中筛选摇滚候选，再对候选详情页进行有上限的补充访问，结合艺人和详情页证据识别是否为后摇演出。
- 命中后摇后生成待同步后台清单；后台未配置时不得真实上架。
- 图片只在最终同步后台阶段抓取。
- 交付为 Codex 可安装插件目录。
- 支持人工触发和定时入口。

## 样例验收

在线采集入口：

```text
https://www.showstart.com/
```

在线列表采集期望：

| 字段 | 期望 |
|---|---|
| 采集数量 | 至少 `5` 条去重记录；数量由 `max_items` 控制 |
| 列表字段 | 每条记录包含 `event_name`、`artist_or_subject`、`detail_url`、`list_position`、`card_evidence` 或明确字段缺失诊断 |
| 去重 | 以规范化详情 URL 或演出名+日期等稳定键去重，并报告抓取数、去重后数量和跳过原因 |
| 摇滚筛选 | 输出所有摇滚候选及其列表证据；不能只输出一条结果 |
| 详情补充 | 仅访问候选详情页，`detail_limit` 可配置并记录未访问原因 |
| 同步行为 | 后台未配置时只生成多条记录的 dry-run `sync_plan`，不得真实上架或抓图 |

离线分类样例（用于稳定验证分类逻辑，不代表在线列表页一定出现该演出）：

| 字段 | 样例值 | 期望 |
|---|---|---|
| 演出名 | `【云南后摇新锐】 一便士草莓地 新专辑《Wasteland Music》专场` | 不是站点通用标题 |
| 艺人 | `一便士草莓地` | 非空 |
| 风格证据 | `后摇` 或 `post-rock` | 证据非空 |
| 分类结果 | 后摇候选 | 进入候选清单 |

如果网络或权限阻止访问，不得伪造上述字段，必须输出 `network_permission_denied`、`timeout`、`http_error`、`captcha_or_login`、`selector_miss` 或等价诊断，并给出复跑建议。

## 必须检查

- 插件根目录包含 `.codex-plugin/plugin.json`、`skills/` 和 `skills/{SKILL_NAME}/SKILL.md`。
- 文档包含安装、重装、源码修正后同步已安装缓存和安装后结构检查说明。
- `run_daily_check.ps1` 支持可配置 Python 路径，不能只写裸 `python`。
- `scripts/smoke_test.py` 或 `scripts/run_business_use_case_test.py` 包含样例期望结果校验，不只检查产物文件存在。
- `references/business-use-case-profile.json` 记录样例 URL、期望字段、期望分类、dry-run 同步约束和失败诊断要求。
- 成品目录不包含 `__pycache__/`、`*.pyc`、`work_smoke/`、`tmp_smoke/` 或业务测试报告残留。

## 通过标准

- 创建器用例评分不低于 85。
- 无 P0/P1。
- 若真实访问列表页，能输出至少 5 条去重记录、列表字段质量结果、摇滚候选和多条 dry-run 同步计划。
- 若在线列表页暂时没有后摇演出，不能伪造命中；离线分类样例仍须验证后摇候选逻辑。
- 若访问失败，必须输出明确网络/权限/页面诊断，且当前成熟度不得高于 L3 可执行脚手架。
